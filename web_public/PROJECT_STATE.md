# Captain of Industry — Public Web Planner: Состояние проекта

## Структура репозитория

```
capitain of industry/
├── web/                        # Локальная версия (не трогать без явного указания)
│   ├── app.py                  # Flask, порт 5000
│   ├── templates/
│   │   ├── index.html          # Главная: Browse рецептов/комплексов
│   │   └── complex_editor.html # Редактор комплексов
│   └── static/icons/           # PNG иконки ресурсов
└── web_public/                 # Публичная версия (основная работа здесь)
    ├── app.py                  # Flask, порт 5001
    ├── auth.py                 # Google OAuth, Steam OpenID, session code, guest UUID
    ├── db.py                   # psycopg2 connection helper
    ├── i18n/
    │   ├── en.json             # Английские переводы
    │   └── ru.json             # Русские переводы
    ├── sql/
    │   └── schema.sql          # Монолитный DDL (единственный файл для новых стендов)
    ├── templates/
    │   ├── index.html          # Главная
    │   └── complex_editor.html # Редактор
    └── static/icons/ -> ../web/static/icons/  # junction link
```

## База данных

- **БД:** PostgreSQL 14, база `coi_public`
- **Подключение:** `postgresql://postgres:postgres@127.0.0.1:5432/coi_public`
- **Игровые данные** (рецепты, рецепты, здания) — уже в БД, берутся из `web/`
- **Развернуть с нуля:** `psql -U postgres -d coi_public -f web_public/sql/schema.sql`

### Ключевые таблицы

| Таблица | Назначение |
|---------|-----------|
| `users` | Пользователи. `provider`: google/steam/session_code/guest. `is_guest=TRUE` + `guest_cookie UUID` для гостей |
| `session_codes` | Коды входа (fallback без OAuth) |
| `user_settings` | key-value настройки (ui_language, ui_theme, show_public_complexes) |
| `user_recipe_prefs` | Скрытые рецепты пользователя (`hidden=TRUE`) |
| `complexes` | `user_id`, `visibility` (private/public), `slug UUID`, `likes_count`, `forked_from_id` |
| `complex_likes` | Лайки: (user_id, complex_id) PK |
| `resource_flows` | Потоки ресурсов через рецепты и комплексы |

## Текущая функциональность

### Главная страница (index.html)

**Табы:** Рецепты | Сообщество

**Вкладка Рецепты:**
- Показывает ВСЕ рецепты + комплексы текущего пользователя (user_id совпадает)
- Комплексы с `user_id=NULL` (мигрированные из локальной версии) — НЕ показываются
- Поиск с поддержкой точного совпадения (`"text"`)
- Фильтры: Все / Рецепты / Комплексы / чекбокс Скрытые
- Колонки: Здание/Комплекс, Рабочие, КВт, Обслуживание, Вход, Выход, кнопки действий
- Кнопки на строке: граф-иконка (создать комплекс на основе), ✏️ (редактировать комплекс), 🗑 (удалить с подтверждением), 👁/🙈 (скрыть/показать)
- Batch-операции: выбор строк → скрыть/показать/удалить (удаление только если все выбранные — комплексы)
- Пагинация (50 записей/страница)

**Вкладка Сообщество:** отображение публичных комплексов (UI заготовлен, логика есть)

**Шапка:** Рецепты | Сообщество | поиск | EN/РУ | 🌙/☀️ | Войти (для гостей) / аватар+меню (для залогиненных)

### Редактор комплексов (complex_editor.html)

- **Создание:** только через кнопку граф-иконки на строке рецепта/комплекса (`/complex/new?from_type=recipe&from_id=X`)
- **Холст:** drag-and-drop узлов, соединения стрелками
- **Узлы:** рецепты и комплексы, счётчик зданий, порты входа/выхода
- **Пикер:** открывается кликом по порту. Фильтры: Все / Рецепты / Комплексы / Сообщество. Вкладка Сообщество — топ-10 публичных комплексов с нужным ресурсом, сортировка по лайкам
- **Сохранение:** две кнопки — «Сохранить» (остаёмся в редакторе) и «Сохранить и выйти». Предупреждение при попытке уйти с несохранёнными изменениями
- **Видимость:** приватный/публичный. При попытке поставить публичный без авторизации — модал с предложением войти
- **Гостевое уведомление:** при первом «Сохранить и выйти» показывается модал с рекомендацией зарегистрироваться
- **Футер:** Итого: рабочие / КВт / обслуживание | Вход: ресурсы | Выход: ресурсы
- **Без auth-UI в шапке** редактора (только язык и тема)

### Авторизация

- **Гостевой UUID:** каждый посетитель автоматически получает guest-аккаунт (httponly cookie `coi_guest`, 1 год). Гость может скрывать рецепты и создавать комплексы
- **Google OAuth:** настроен через `authlib` (требует env vars)
- **Steam OpenID 2.0:** настроен вручную (требует `STEAM_API_KEY`)
- **Session code:** fallback без OAuth, 8-символьный код
- **Merge гостя при логине:** TODO (см. план)
- **Гость НЕ может:** публиковать комплексы, форкать чужие

### i18n

- Сервер-сайд: функция `t('key.subkey')` в Jinja2 шаблонах
- Клиент-сайд: `jt('key', {vars})` через `window._i18n`
- Языки: EN / РУ. Переключение через query param `?lang=ru`
- Dev-сервер перезапускается при изменении JSON (via `extra_files` в `app.run`)

### Тема

- localStorage (`coi_theme`: 'dark'/'light')
- Tailwind dark mode через CSS класс на `<html>`
- Инициализация без мигания: inline script в `<head>` до загрузки Tailwind

## Ключевые архитектурные решения

| Решение | Что выбрано |
|---------|------------|
| Скрытие рецептов | `user_recipe_prefs` в БД (не localStorage) |
| URL комплексов | UUID slug (не числовой ID, защита от перебора) |
| Unique имя комплекса | Per-user `(user_id, name)` index, не глобальный |
| Deprecated рецепты | Удалено — только user-hide через prefs |
| Новый комплекс | Только через "создать на основе рецепта/комплекса", не пустой холст |
